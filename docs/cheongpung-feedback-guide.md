# 강화유니버스 피드백 참여 가이드

> 청풍 관계자분들이 배포 링크를 직접 둘러보며 사용자, 사장님, 크루 입장에서 피드백을 남기기 위한 안내문입니다.

## 0. 먼저 접속하기

- 배포 링크: `https://ganghwa-universe.vercel.app`
- 권장 환경: 모바일 Safari/Chrome 또는 데스크톱 Chrome
- 테스트용 코드: 사장님 코드와 크루 코드는 별도 메시지로 전달받은 값을 사용해 주세요.

이 웹앱은 강화도 안에서 쌓이는 참여자, 가게, 프로젝트의 관계를 카드와 공개 화면으로 보여주는 초기 모델입니다. 아직 최종 운영 버전이 아니라, “무엇이 잘 이해되는지”, “어디서 헷갈리는지”, “청풍이 실제로 운영하기에 어떤 부분이 더 필요할지”를 확인하는 용도입니다.

## 1. 전체적으로 봐주실 것

아래 질문을 떠올리며 둘러봐 주세요.

- 첫 화면만 보고 “강화유니버스가 무엇을 하는 서비스인지” 이해되는가?
- 참여자, 사장님, 크루가 각각 무엇을 하면 되는지 구분되는가?
- 강화도에 관계와 활동이 쌓인다는 느낌이 전달되는가?
- 청풍이 실제 운영할 때 불편하거나 위험해 보이는 부분이 있는가?
- 최종 발표 전에 꼭 고쳐야 할 부분과, 이후 개선해도 되는 부분은 무엇인가?

## 2. 사용자 입장에서 보기

사용자는 강화도 프로젝트에 참여하고, 방문이나 활동의 흔적을 카드로 남기는 사람입니다.

### 따라 해보기

1. 배포 링크 첫 화면에 접속합니다.
2. 상단 메뉴에서 `임팩트`, `프로젝트`, `가게`, `피드`를 차례로 눌러봅니다.
3. `/impact` 화면에서 강화유니버스 현황과 관계 노드맵을 확인합니다.
4. `/projects`에서 프로젝트가 4개 카테고리로 잘 나뉘는지 봅니다.
5. `/shops`에서 가게 목록과 가게 상세 화면을 확인합니다.
6. `/feed`에서 공개된 카드들이 어떻게 보이는지 확인합니다.
7. 참여자 카드 작성 링크 또는 QR을 전달받았다면, 카드 작성 화면에서 메모를 남겨봅니다.
8. 카카오 로그인이 가능하다면 `/collection`에서 내가 남긴 카드가 도감처럼 보이는지 확인합니다.

### 피드백 포인트

- 카드 작성 흐름이 어렵거나 부담스럽지는 않은가?
- 공개/비공개 선택이 충분히 이해되는가?
- 내 도감이 “내가 강화에 남긴 것”처럼 느껴지는가?
- 공개 피드와 임팩트 화면이 외부 사람에게 보여줘도 괜찮은 톤인가?

## 3. 사장님 입장에서 보기

사장님은 본인 가게에 연결된 참여자 카드들을 보고, 참여자에게 짧은 편지를 남길 수 있습니다.

### 따라 해보기

1. 상단 메뉴의 `운영 로그인`에서 `사장님 로그인`을 선택합니다.
2. 별도로 전달받은 가게 주소와 8자리 코드를 입력합니다.
3. 로그인 후 `/owner` 화면에서 본인 가게에 연결된 참여자 카드를 확인합니다.
4. 카드 하나를 골라 편지 작성 화면으로 들어갑니다.
5. `AI 첫 문장 제안`을 눌러 초안이 만들어지는지 확인합니다.
6. 초안을 그대로 보내지 말고, 사장님 말투로 수정한 뒤 저장합니다.
7. 참여자 카드 상세 화면에서 사장님 편지가 잘 보이는지 확인합니다.

### 피드백 포인트

- 가게 주소와 코드 입력 방식이 충분히 쉬운가?
- 어떤 카드에 답장해야 하는지 바로 이해되는가?
- AI 초안 안내가 “참고만 하고 직접 수정해야 한다”는 느낌을 주는가?
- 사장님이 실제로 쓰기에 편지 작성 부담이 줄어드는가?
- 사장님에게 보여주면 안 될 정보가 보이지는 않는가?

## 4. 크루 입장에서 보기

크루는 현장에서 프로젝트와 에피소드 진행 상태를 확인하고, 참여 카드에 간단한 반응을 남기는 역할입니다.

### 따라 해보기

1. 상단 메뉴의 `운영 로그인`에서 `크루 로그인`을 선택합니다.
2. 별도로 전달받은 크루 코드를 입력합니다.
3. `/crew` 화면에서 예정, 진행 중, 완료 에피소드를 확인합니다.
4. 에피소드 상태를 변경할 수 있는지 확인합니다.
5. 상태 변경 후 프로젝트 상세 화면에 진행 상태가 자연스럽게 반영되는지 확인합니다.
6. 최근 카드에 하이파이브나 짧은 노트를 남겨봅니다.

### 피드백 포인트

- 크루가 현장에서 바로 쓰기엔 화면이 충분히 단순한가?
- 에피소드 상태 이름이 실제 운영 언어와 맞는가?
- 크루가 할 일과 청풍 관리자만 해야 할 일이 잘 나뉘는가?
- 실수로 잘못 누를 위험이 있어 보이는 버튼이나 흐름이 있는가?

## 5. 피드백 남기는 방식

피드백은 아래 형식으로 남겨주시면 정리하기 쉽습니다.

| 구분 | 예시 |
| --- | --- |
| 역할 | 사용자 / 사장님 / 크루 / 전체 |
| 화면 | `/impact`, `/shops`, `/owner` 등 |
| 상황 | 어떤 순서로 보다가 느꼈는지 |
| 의견 | 헷갈림, 좋아 보임, 어색함, 빠진 것 |
| 우선순위 | 발표 전 필수 / 발표 후 개선 / 보류 |

짧게 남겨도 괜찮습니다. 예를 들어 “사장님 로그인에서 가게 주소가 뭔지 모르겠다”, “프로젝트 카테고리 이름은 좋은데 설명이 더 있으면 좋겠다”처럼 실제로 멈칫한 지점을 알려주시면 가장 도움이 됩니다.

## 6. 주의할 점

- 테스트 중 입력한 메모나 사진은 실제 배포 환경에 남을 수 있습니다.
- 개인정보, 민감한 사진, 외부 공개가 어려운 내용은 입력하지 말아 주세요.
- 사장님 코드와 크루 코드는 외부에 공유하지 말아 주세요.
- 오류가 나면 화면 주소, 어떤 버튼을 눌렀는지, 가능하면 스크린샷을 함께 남겨 주세요.
