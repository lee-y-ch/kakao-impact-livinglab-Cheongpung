# 강화유니버스 팀원 체험 가이드

> 목적: 팀원들이 배포된 웹앱을 직접 눌러보며 현재 활성화된 기능을 전체적으로 경험하고, 회의 전 피드백을 모으기 위한 안내서입니다.

## 1. 접속 주소

- 배포 URL: `https://ganghwa-universe.vercel.app`
- 최신 Vercel 배포 URL: `https://ganghwa-universe-fnqy020iy-lychs-projects.vercel.app`

둘 중 하나로 접속하면 됩니다. 보통은 짧은 배포 URL을 먼저 쓰고, 접속이 이상하면 최신 Vercel 배포 URL로 확인합니다.

## 2. 준비물

- 참여자 체험: 카카오 로그인 가능한 계정
- 사장님 체험: 별도 공유받은 가게 주소와 8자리 코드
- 크루 체험: 별도 공유받은 크루 공용 코드
- 관리자 체험: 별도 공유받은 관리자 계정

코드와 비밀번호는 이 문서에 적지 않습니다. 팀 메신저의 별도 안전한 채널에서 확인해 주세요.

## 3. 전체 체험 순서

처음 보는 팀원은 아래 순서대로 보면 앱의 전체 구조를 가장 빠르게 이해할 수 있습니다.

1. 공개 랜딩
2. 공개 임팩트
3. 프로젝트와 가게 둘러보기
4. 참여자 카드 작성과 내 도감
5. 사장님 편지
6. 크루 운영
7. 관리자 운영

## 4. 공개 화면 둘러보기

### 4.1 랜딩

경로: `/`

확인할 것:

- 첫 화면에서 강화유니버스가 어떤 프로젝트인지 이해되는지
- “강화유니버스 현황” 3개 카드 hover 색이 자연스러운지
- 프로젝트 카드와 공개 카드 갤러리가 보기 좋은지
- 상단 바의 `운영 로그인`에서 사장님/크루/관리자 로그인으로 갈 수 있는지

### 4.2 임팩트

경로: `/impact`

확인할 것:

- 누적 카드, 가게, 참여자, 에피소드, 사장님 편지 숫자가 보이는지
- 노드맵에서 프로젝트, 가게, 참여자가 연결되어 보이는지
- 노드에 마우스를 올렸을 때 카드 미리보기가 뜨는지
- `전체` 모드와 `발표` 모드를 바꿔보며 어느 쪽이 회의/발표에 더 좋은지

### 4.3 프로젝트

경로: `/projects`

확인할 것:

- 4개 카테고리가 이해되는지
- `시부야대학 교류` 프로젝트 상세에 들어가 에피소드와 공개 카드가 자연스럽게 이어지는지
- 프로젝트 설명이 청풍의 언어와 맞는지

### 4.4 가게

경로: `/shops`

확인할 것:

- 가게 목록이 잘 보이는지
- 가게 상세에서 공개 카드와 가게 분위기가 이어지는지
- 참여 가게가 더 추가되어야 할 것 같은지

### 4.5 피드

경로: `/feed`

확인할 것:

- 공개 카드가 시간순으로 자연스럽게 보이는지
- 카드 문장과 카테고리 배지가 어색하지 않은지
- 비공개일 것 같은 내용이 공개되어 보이지 않는지

## 5. 참여자 체험

### 5.1 로그인

경로: `/login`

카카오 로그인으로 들어갑니다. 로그인 후 `/collection`으로 이동하는지 확인합니다.

### 5.2 카드 작성

경로: `/entry/demo-gyodong-bookshop`

해볼 것:

- 메모만 입력하고 저장
- 가능하면 사진 첨부도 테스트
- 공개 여부를 켜고 끄며 문구가 이해되는지 확인
- 저장 후 `/collection`에 카드가 생기는지 확인

### 5.3 내 도감

경로: `/collection`

해볼 것:

- 내 카드 목록 확인
- 상단 4개 요약 카드 클릭
- 카드 상세 진입
- 공개/비공개 토글
- 사장님 편지나 하이파이브가 카드 상세에 보이는지 확인

## 6. 사장님 체험

상단 바 `운영 로그인` → `사장님 로그인` 또는 경로 `/owner/login`

해볼 것:

- 별도 공유받은 가게 주소와 8자리 코드로 로그인
- 코드 입력칸의 보이기/숨기기 버튼 사용
- `/owner`에서 내 가게에 연결된 카드 확인
- 대표 카드로 편지 쓰기 진입
- `AI 첫 문장 제안` 버튼 사용
- 초안을 직접 수정해서 편지 저장
- 참여자 카드 상세에서 편지가 보이는지 확인

확인할 것:

- 사장님이 이 화면만 보고도 무엇을 해야 하는지 알 수 있는지
- AI 초안을 그대로 보내지 말고 수정해야 한다는 안내가 충분한지
- 편지 쓰기 흐름이 부담스럽지 않은지

## 7. 크루 체험

상단 바 `운영 로그인` → `크루 로그인` 또는 경로 `/crew/login`

해볼 것:

- 별도 공유받은 크루 코드로 로그인
- `/crew`에서 예정/진행/완료 에피소드 확인
- 에피소드 상태 변경
- 최근 카드에 하이파이브 또는 note 남기기
- 프로젝트 상세에서 상태가 반영되는지 확인

확인할 것:

- 크루가 현장에서 직접 쓸 수 있을 정도로 단순한지
- 어떤 작업은 크루가 아니라 관리자가 해야 할 것 같은지

## 8. 관리자 체험

상단 바 `운영 로그인` → `관리자 로그인` 또는 경로 `/admin/login`

해볼 것:

- 관리자 계정으로 로그인
- `/admin` 운영 홈 확인
- `/admin/projects`에서 프로젝트와 에피소드 구조 확인
- `/admin/shops`에서 가게, QR, 사장님 코드 발급 위치 확인
- `/admin/review`에서 공개 검수 큐 확인
- `/admin/reports`에서 신고 처리 큐 확인

확인할 것:

- 청풍 담당자가 직접 운영할 수 있는 수준인지
- 메뉴 이름이나 설명이 더 쉬워져야 하는지
- 실수로 잘못 공개하거나 삭제할 위험이 있어 보이는지

## 9. 피드백 남길 때 봐야 할 기준

피드백은 아래처럼 남겨주면 정리하기 쉽습니다.

| 구분 | 예시 |
| --- | --- |
| 화면 | `/impact`, `/owner/letters/new`, `/collection` |
| 역할 | 참여자 / 사장님 / 크루 / 관리자 / 공개 방문자 |
| 문제 또는 의견 | 무엇이 헷갈렸는지, 어떤 문구가 어색했는지 |
| 중요도 | 발표 전 필수 / 회의 후 개선 / 나중에 해도 됨 |
| 제안 | 가능하면 어떻게 바꾸면 좋을지 |

## 10. 꼭 봐줬으면 하는 질문

- 처음 들어왔을 때 이 서비스가 무엇인지 이해되는가?
- 참여자 카드가 “강화에 환대를 남기는 기록”처럼 느껴지는가?
- `/impact`의 `전체` 모드와 `발표` 모드 중 어떤 화면이 설명하기 좋은가?
- 사장님 편지 흐름이 실제 사장님에게 부담스럽지 않은가?
- 관리자 화면은 청풍 담당자가 직접 운영할 수 있을 만큼 단순한가?
- 발표 전에 꼭 고쳐야 할 화면이나 문구가 있는가?
