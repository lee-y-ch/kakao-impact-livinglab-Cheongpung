## TL;DR

- **주제** : *"만남에서 태어난 카드가 시간이 흘러 편지로 돌아오는 웹 경험"*
- 청풍 **Q1(관계 시각화) + Q2(응원의 편지)** 를 하나로 결합
- 매체는 **모바일 웹/PWA**, 네이티브 앱 아님
- 4/11 현장에서 6개 가설 검증 → 4/12 주제 확정

## 왜 이 방향인가 (3줄)

1. 3/30 대표님 미팅 원칙 : *"플랫폼 말고 관계 중심, 어떻게 오프라인 관계를 유지할 것인가"*
2. 대표님 최대 고통 : **"보이지 않는 성과"** → 관리자 뷰로 직접 해결
3. 개발자 1명 + 6월까지 + 이을랩 앱과 차별화 → 웹 경험 레이어가 가장 현실적

## 만들 것 (3개)

| 구성 | 내용 | 대응 |
|---|---|---|
| **① 만남 카드** | 오프라인에서만 발급. 뒷면 한 줄 기록 필수. 같은 상대는 카드 1장이지만 뒷면이 두꺼워짐 | 청풍 Q1 |
| **② 응원의 편지** | 떠난 뒤 N주~N개월 후 시차 발송. LLM 문구 보조 + 사람 최종 손길 | 청풍 Q2 |
| **③ 관계 밀도 도감** | 개인 회고 화면 + **청풍 관리자 뷰** (대표님 보고서용) | Q1 확장 |

## 절대 어기면 안 되는 4가지

1. **카드 발급 트리거는 무조건 오프라인** — 앱 안 "뽑기 버튼" 금지. 어기면 플랫폼화
2. **네이티브 앱 금지** — 웹/PWA. 이을랩 충돌 + 개발자 1명 한계
3. **게임성 4규칙** : ① 거래 불가 ② 1관계=1카드 ③ 도감 기본 비공개 ④ 뒷면 한 줄 없으면 저장 불가
4. **청풍이 6월 이후 직접 운영 가능해야** — 관리자 UX 단순화 우선

## 4/11 현장에서 검증할 6개 가설

| # | 가설 | 무너지면 |
|---|---|---|
| H1 | 떠난 뒤 연락 못한 경험이 참가자에게 실제로 있다 | 편지 메커니즘 수요 없음 |
| H2 | 상인이 월 1~2회 편지 쓰기 가능 | 청풍 대리 작성 모델로 전환 |
| H3 | 가게 QR/NFC 부착 거부감 없음 | 트리거 방식 재설계 |
| H4 | 관리자 뷰가 "보이지 않는 성과" 문제를 실제로 해결 | 방문자 단방향 서비스로 축소 |
| H5 | 재방문 트리거가 "사람"이다 | 장소/경험 중심으로 피벗 |
| H6 | 기존 강화유니버스 자산과 연결 접점 있음 | 통합 재검토 |

**하나라도 무너지면 현장에서 재설계.**

## 4/11 현장 질문지

### 대표님께 (최우선, 순서대로)

1. Q1(관계의 시각화)에서 머릿속에 그리는 구체적 장면
2. "보이지 않는 성과" — 보고서에 들어가야 할 숫자/장면은 구체적으로?
3. **편지 발신자는 누구?** (상인/청풍/대표/참가자) ← **이 답에 따라 설계 전체 달라짐**
4. 이을랩 앱과 겹치면 안 되는 선
5. 지금도 개인적으로 연락 이어지는 참가자 수
6. 6월 이후 청풍 운영 의향
7. 오프라인 설치 가능 공간 (공용 스페이스)

### 상인·주민께 (가게 돌면서)

- 한달살기 참가자 중 지금도 연락하는 분 있는지
- 안부 편지 쓸 의향 (월 1~2회 가능?)
- QR/NFC 스티커 부착 거부감
- **(관찰)** 가게 내 이미 있는 관계 흔적(사진·방명록·포스트잇) 전부 촬영

### 잠시섬 참가자께 (4/11 별사냥 프로그램)

- 떠난 뒤 연락하고 싶었는데 못한 경험
- 재방문 시 누구를 먼저 보러 오고 싶은지 — **장소 말고 사람**
- 강화의 인상 깊은 순간을 카드 한 장으로 남긴다면?
- 한 달 뒤 편지 도착 → 반가움 vs 부담
