## 한 줄 정체성

> **오늘도 강화도가 조금씩 더 강화됩니다.**
>
> 참여자·크루·사장님의 환대 행위가 쌓여 강화도의 서사가 되는 **관계 대시보드**.
> 모바일 웹 (PWA) — 설치 없이 브라우저에서 바로 사용. 네이티브 앱 아님.

## 중간 발표 이후 무엇이 바뀌었나

1. 피드백 "주인공이 모호하다" → **세 뷰를 유지하되 발표 내러티브는 참여자 카드 중심**
2. 기존 `/admin` 이 "청풍 노드맵 시각화 + 운영" 이었는데 → **"강화도가 얼마나 강화됐나"는 공개 페이지로 빼고, `/admin` 은 순수 운영 도구** 로 분리
3. **크루 역할 명확화** : 에피소드 진척 업데이트 담당 → 크루 로그인을 Phase 4 → **Phase 3** 로 당김

## 세 주인공의 뷰 (재정의)

| 뷰 | 주인공 | 메인 페이지 | 핵심 질문 |
| --- | --- | --- | --- |
| **참여자 뷰** | 나 (카카오 로그인) | 내 도감 | "내가 강화에 무엇을 쌓았나?" |
| **강화도 뷰 (공개)** | 강화도 자체 | **임팩트 페이지 — 누구나** | "강화도가 얼마나 강화됐나?" |
| **운영 뷰** | 청풍 + 크루 | 운영 대시보드 | "미션 올리고 진행 업데이트" |

**핵심 변화** : "강화도가 얼마나 강화됐나" 는 **청풍 내부 지표가 아니라 공개 서사**. 대표님의 "보이지 않는 성과" 문제는 외부에 직접 보여주는 쪽이 더 강하게 해결됨.

## 4가지 카테고리 (청풍 정의)

|   | 카테고리                      | 의미                                                  |
| - | ----------------------------- | ----------------------------------------------------- |
| 1 | **환대의 공유지** (commons)    | 강화도 안의 공유 자원·공간·관계 연결                    |
| 2 | **환대의 네트워크** (network)  | 강화 안팎 사람·조직 교류·협업 (예: 시부야대학)          |
| 3 | **환대의 세계** (world)        | 환대를 담은 콘텐츠·브랜드·작품                          |
| 4 | **환대의 정책** (policy)       | 환대 관점 지역 정책·제도 제안                           |

```
카테고리 (4개)
  └─ 프로젝트 (장기, 여러 해)
        └─ 에피소드 (회차/세션, 특정 날짜)
              └─ 카드 (참여자 한 명이 한 번 남긴 기록)
```

---

## 핵심 개념 — "카드가 쌓이면 서사가 된다"

- **카드 1장** = 참여자가 남긴 환대 행위 1건 (사진 + 한 줄 + 타입)
- 같은 에피소드에 여러 번 참여하면 카드 여러 장 — 반복 자체가 의미
- 카드 앞면 = 참여자가 쓴 것 / 뒷면 = 크루·사장님이 달아준 **응원·편지·하이파이브**

### 카드 타입

| 타입         | 의미                       | 주로 누가    |
| ------------ | -------------------------- | ------------ |
| 메모         | 한 줄 기록                 | 참여자       |
| 사진         | 사진 + 캡션                | 참여자       |
| 체크인       | QR 스캔 / 장소 방문 기록   | 참여자       |
| 워크숍       | 워크숍 참여·산출물         | 참여자       |
| 하이파이브   | 짧은 응원 카운터           | 누구나       |
| 결과물       | 에세이·작품·기록물 제출    | 참여자·크루  |
| 아카이브 링크| 기존 아카이브 URL 등록     | 크루         |

## 지켜야 할 원칙

1. **카드·포인트는 거래 불가** — 계정 귀속
2. **공개는 기본 꺼짐 (opt-in)**
3. **사진·메모 둘 다 비면 저장 불가** — 빈 카드 방지
4. **경쟁·중독 메커니즘 금지** — 좋아요, 팔로우, 공개 댓글, 공개 랭킹, 개인 점수 노출 없음
   - 단 카테고리·프로젝트·가게 필터, 관리자 검색, 편지는 허용 ("발견 가능성"과 "경쟁 유도"는 분리)
5. **개인정보·초상권 보호** — 얼굴·타인 포함 자가 체크, 신고·삭제 절차

## 랜딩 — 누가 처음에 뭘 보나

홈(`/`) 접근 시 로그인 상태에 따라 자동 분기.

| 누구 | 처음 보는 화면 |
| --- | --- |
| 로그인 안 한 방문자 | **공개 랜딩** — 강화도 임팩트 요약 + 참여자 카드 히어로 |
| 참여자 | **내 도감** (`/collection`) |
| 크루 | **크루 대시보드** (`/crew`) |
| 사장님 | **우리 가게** (`/owner`) |
| 청풍 관리자 | **운영 홈** (`/admin`) |

→ 참여자 개인 도감은 본인만 접근. 크루·관리자는 공개 페이지(`/impact`, `/feed`, `/projects`)에서 activities 를 본다.

## 세 주인공별 경험 과정 (시간 순서대로)

### ① 단순 방문자 (비로그인, 홍보 링크로 들어옴)

```
[링크 타고 진입]
   ↓
[/ 공개 랜딩]
   • 상단 : 강화도 임팩트 요약 카드 ("카드 284 · 가게 12 · 응원 97")
   • 중단 : 참여자 카드 갤러리 히어로 — "강화도에서 벌어진 환대"
   • 하단 : 카테고리별 진척 바 요약 + "자세히 보기" → /impact
   ↓
[/impact — 강화도 진척 대시보드]
   "강화도가 얼마나 강화됐나" 를 깊이 볼 수 있는 페이지
   • 실시간 카운터 (카드·가게·참여자·에피소드·응원)
   • 카테고리별 진척 (공유지·네트워크·세계·정책 각각)
   • 진행 중 프로젝트 타임라인 (예: 시부야 교류 3/5회)
   • 노드맵 시각화 (공개 카드만)
   • 최근 공개 카드 미리보기 → /feed 로 이동
   ↓
[/projects — 프로젝트 리스트]
   카테고리별로 장기 프로젝트 보기
   ↓
[/projects/[이름] — 프로젝트 상세]
   • 진척도 바 (기간 / 회차 / 목표 중 팀이 정한 기준)
   • 에피소드 타임라인 (완료 · 진행 · 예정)
   • 이 프로젝트의 공개 카드 모음
   ↓
[/shops, /shops/[id] — 가게 탐색]
   지역 가게별 공개 카드 갤러리
   ↓
[/feed — 커뮤니티 피드]
   전체 공개 카드 시간순
   ↓
[참여하고 싶다]
   → /login → 카카오 로그인 → "참여자" 로 전환
```

**방문자가 볼 수 없는 것** : 개별 참여자 도감, 비공개 카드, 관리자 운영 페이지.

### ② 참여자 (일반 사용자 → 카카오 로그인)

**A. 강화도 현장에서 첫 카드 발급**

```
[가게·에피소드 현장에서 QR 스캔]
   ↓
[/entry/[QR토큰]]
   • 비로그인 상태면 → 카카오 로그인 버튼 표시
   ↓
[/login → 카카오 로그인]
   • 닉네임·프로필 자동 연동
   • 로그인 후 자동으로 /entry/[QR토큰] 로 복귀
   ↓
[카드 작성 화면]
   • 어떤 가게·에피소드인지 자동 표시
   • 사진 1장 (즉석 촬영 가능)
   • 한 줄 메모
   • 얼굴·타인 포함 자가 체크
   • 공개 / 비공개 토글 (기본 비공개)
   ↓
[저장]
   ↓
[/collection 내 도감으로 이동]
   • 방금 등록한 카드가 그리드 첫 번째에 나타남
```

**B. 떠난 뒤 앱을 다시 열면**

```
[브라우저·홈 화면 아이콘으로 / 접근]
   ↓
[자동 분기] → [/collection 내 도감]
   • 카드 그리드 (시간순)
     - 💌 아이콘 = 응원·편지 받은 카드
     - 빈 카드 = 아직 반응 없는 카드
   • 상단 누적 : "카드 12 · 가게 5 · 에피소드 4"
   • 카테고리별 내가 모은 카드 수
   • (개인 점수·순위는 없음)
   ↓
[카드 탭] → [/collection/[id] 카드 상세]
   • 앞면 : 내가 쓴 것 (사진·메모)
   • 뒷면 : 받은 응원·편지·하이파이브
   • 공개 ↔ 비공개 언제든 토글
   • 삭제·신고 요청
   ↓
[프로필 수정] → [/me]
   ↓
[다른 사람들 활동 구경]
   → /impact (강화도 전체 진척)
   → /projects, /projects/[이름] (프로젝트별 진척)
   → /feed (공개 카드 시간순)
   → /shops (가게 탐색)
```

**참여자가 볼 수 있는 페이지** :
- 본인 도감 (`/collection`, `/me`)
- 모든 공개 페이지 (`/impact`, `/projects`, `/shops`, `/feed`)

**참여자가 볼 수 없는 것** : 남의 개별 도감, 운영 페이지, 관리자 내부 지표.

```

   • AI 첫 문장 제안 ("○○님, 그날 ○○ 해주셔서…")
   • "AI 초안이니 꼭 수정해서 보내세요" 라벨 항상 표시
   • 사장님이 자유롭게 편집
   • 공개 / 비공개 선택 → 발송
   ↓
[참여자 도감에 즉시 반영]
   • 참여자는 본인 /collection/[id] 에서 편지 확인
```

**C. 가게 설정**

```
[/owner/settings]
   • 가게 소개 (한 줄)
   • 영업시간
   • 가게 페이지 공개 ON/OFF (기본 ON)
     - OFF 하면 그 가게의 공개 카드도 외부에서 안 보임
```

**사장님이 볼 수 있는 페이지** :
- 우리 가게 관련 (`/owner/*`)
- 모든 공개 페이지 (`/impact`, `/projects`, `/shops/[본인가게]`, `/feed`)

**사장님이 볼 수 없는 것** : 다른 가게 대시보드, 참여자 개별 도감, 운영 페이지.

### ⑤ 청풍 관리자 (운영 책임자)

**A. 처음 로그인**

```
[/admin/login]
   • Supabase Auth (이메일·비밀번호)
   • 계정별 감사 로그 자동
   ↓
[로그인 성공 → /admin 운영 홈으로 이동]
```

**B. 운영 홈 — 오늘 처리할 것들 한눈에**

```
[/admin — 운영 대시보드]
   • 검수 대기 카드 : 참여자가 공개 요청한 카드 N건
   • 신고 대기 : 부적절 사진·메모 신고 M건
   • 등록 대기 : 새 가게 등록 요청 / 사장님 코드 재발급
   • 현재 진행 중 에피소드 요약 (크루 업데이트 상태)
   • (참고) "강화도가 얼마나 강화됐나" 는 /impact 에서 공개로 확인
```

**C. 운영 페이지별 하는 일**

```
[/admin/projects — 프로젝트·에피소드 관리]
   • 새 프로젝트 만들기 (카테고리 지정, 기간 설정)
   • 에피소드 추가 (세션 날짜, 목표 참여자 수 등)
   • 진척도 목표 설정 (시간 기반 / 회차 기반 / 목표 기반)

[/admin/shops — 가게 관리]
   • 새 가게 등록 (이름·주소·소개)
   • QR 코드 생성·다운로드 (현장 부착용)
   • 사장님 코드 발급·재발급

[/admin/review — 공개 검수 큐]
   • 참여자가 공개 요청한 카드 → 승인·반려
   • 초상권·개인정보 문제 있는 카드 비공개 전환

[/admin/reports — 신고 대응]
   • 부적절 사진·메모 신고 리스트
   • 삭제·비공개 조치
```

**D. 강화도 전체 진척을 보고 싶으면** → `/impact` (공개 페이지 그대로). `/admin` 은 공개되지 않는 운영 지표 (검수 대기 건수, 신고 대응률 등) 만 담당.

**관리자가 볼 수 있는 페이지** :
- 모든 운영 페이지 (`/admin/*`)
- 모든 공개 페이지
- (필요 시) 개별 참여자 도감 — 신고 대응 맥락에서만

## 개발 단계 (6월 발표까지) — 순서 재편

| Phase | 내용                                        | 시연 가능한 것                                 |
| ----- | ------------------------------------------- | ---------------------------------------------- |
| 0 ✅   | 기본 셋업                                   | 로그인·홈 기본 동작                            |
| 1     | 인증 3종 + 공통 레이아웃                    | 참여자·사장님·관리자 각각 로그인               |
| 2     | **카드 발급 (참여자 뷰 핵심)**              | QR → 사진·메모 → 카드 저장                     |
| 3     | 도감 + **관리자 기본 운영** + **크루 로그인** | 카드 보기 + 프로젝트·가게 등록 + 크루 진척 업데이트 |
| 4     | 크루·사장님 응원·편지 (AI 보조)             | 편지가 참여자 도감에 도착                      |
| 5     | **공개 임팩트 페이지** (`/impact`)           | 누구나 "강화도가 얼마나 강화됐나" 볼 수 있음   |
| 6     | 공개 둘러보기 (프로젝트·가게·피드)         | 비로그인 방문자 전체 흐름                      |
| 7     | 관리자 고도화 + 크루 개인 계정              | 청풍이 일상 운영 가능                          |
| 8     | 정책 점검 + 데모 준비                       | 6월 최종 발표                                  |

**변경점** :
- 크루 로그인 Phase 4 → **Phase 3** (진척도 업데이트가 MVP 핵심)
- Phase 5 가 "청풍 내부 노드맵" → **공개 임팩트 페이지** 로 재정의

### 2. 운영 결정권자

| 영역 | 담당 |
| --- | --- |

### 3. 경쟁/중독 금지 원칙의 구체 범위
- 좋아요·팔로우·공개 댓글·공개 랭킹 → **금지** (확정)
- 개인 점수 노출 → **금지** (확정)
- 필터 / 관리자 검색 → **허용** (확정)
- 참여자 → 사장님 **답장** → MVP 자리만? 정식 기능?

### 4. **진척도의 정의**  ← 추가

`/impact` 와 `/projects/[이름]` 의 진척 바를 뭐로 계산할지.

- **시간 기반** : 프로젝트 기간 대비 현재 위치 (예: "2024–2028 중 3년차")
- **이벤트 기반** : 전체 N회 회차 중 M회 완료 (예: "시부야 교류 3/5회")
- **목표 기반** : 누적 카드·참여자 수가 설정 목표 대비 몇 % (예: "카드 200장 목표 중 145장")
- **혼합** : 프로젝트별로 다른 기준 (관리자가 `/admin/projects` 에서 각 프로젝트마다 선택)

추천은 **혼합** — 프로젝트 성격이 다양해서 하나로 통일하기 어려움.

### 5. 크루 역할 확정
- 현장 관찰 결과 : 참여자 본인이 카드 입력하는 비율이 높음
- 크루의 주 역할은 **에피소드 진척 업데이트 + 참여자 카드 응원** 으로 확정됨
- 추가로 **아카이브 링크 / 결과물 업로드** 도 담당?
- Phase 3 에 공용 코드 하나로 시작 → Phase 7 에 개인 계정
