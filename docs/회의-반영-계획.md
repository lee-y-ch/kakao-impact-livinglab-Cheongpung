#### 이미지 EXIF 회전 + 1024px 압축 — 코드 검증 완료 + 모바일 테스트 절차

**코드 검증** (`src/lib/utils/image.ts`):
- ✅ EXIF orientation: `createImageBitmap(file, { imageOrientation: "from-image" })` 로 자동 보정.
- ✅ 1024px 캡: 긴 변 기준 `MAX_DIMENSION=1024` 비례 축소.
- ✅ JPEG 0.82 인코딩, 20MB 입력 거부, `bitmap.close()` 정리.
- 코드 자체는 발표 기준으로 충분.

**모바일 실기기 1회 테스트 절차** (사용자 액션):
1. 로컬 dev 서버 실행 — `npm run dev` (포트 3001), 개발 PC 와 같은 Wi-Fi.
2. PC IP 확인 후 모바일 브라우저에서 `http://<PC-IP>:3001/entry/demo-shop` 진입 → 카카오 로그인.
3. 카드 작성 폼에서 카메라로 **세로 사진** 1장 촬영, 1장은 가로 사진 — 둘 다 업로드.
4. 검증 포인트:
   - 데스크탑 `/collection` 에서 두 사진 모두 **회전 없이 똑바로** 보이는가 (특히 iOS Safari 카메라).
   - Supabase Dashboard → Storage 에서 업로드된 파일 크기가 **수백 KB 이하** (원본 수 MB 가 아닌가).
   - 가로/세로 비율이 무너지지 않았는가.
5. 둘 다 통과 → 항목 종결. 회전 깨지면 → `imageOrientation: "from-image"` 가 해당 브라우저에 안 먹는 것이므로 별도 수동 회전 보정 필요(Phase 8 에서 다룸).

#### Vercel preview / prod 환경변수 분리 — 매트릭스

배포 시 Vercel Dashboard → Project Settings → Environment Variables 에 다음과 같이 등록.
**Production 과 Preview 는 같은 키지만 다른 값** — 가장 안전한 방식은 **별도 Supabase 프로젝트 2개(dev / prod) 분리**.

| 변수 | Development (로컬) | Preview (PR 미리보기) | Production |
|---|---|---|---|
| `NEXT_PUBLIC_SUPABASE_URL` | dev 프로젝트 URL | dev 프로젝트 URL | prod 프로젝트 URL |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | dev anon | dev anon | prod anon |
| `SUPABASE_SERVICE_ROLE_KEY` | dev service role | dev service role | prod service role |
| `KAKAO_CLIENT_ID` | 카카오 dev 앱 | 카카오 dev 앱 | 카카오 prod 앱 |
| `KAKAO_CLIENT_SECRET` | 카카오 dev 앱 | 카카오 dev 앱 | 카카오 prod 앱 |
| `ANTHROPIC_API_KEY` | dev 키(낮은 한도) | dev 키 | prod 키 |
| `CREW_ACCESS_CODE` | 임의 | preview 용 | prod 용 (청풍 전달) |
| `NEXT_PUBLIC_SITE_URL` | `http://localhost:3001` | `https://*-preview.vercel.app` (preview 도메인) | `https://<프로젝트도메인>` |

**원칙**:
- **Preview 가 prod DB 를 절대 건드리지 않게** — 별도 Supabase 프로젝트가 가장 단순한 격리.
- Anthropic 키는 사용량 한도 분리(개발 중 폭주 방지).
- `NEXT_PUBLIC_SITE_URL` 은 카카오 OAuth 콜백·앱 메타에 사용되므로 환경별 정확한 값 필수.

(사용자: 실제 prod 배포 시작 시점에 Vercel 대시보드에서 적용)

#### 카카오 OAuth Redirect URI prod 등록
- **무엇**: 카카오 디벨로퍼스 → 내 애플리케이션 → 카카오 로그인 → Redirect URI 에 prod 도메인 추가. Supabase Auth → Providers → Kakao 도 같은 도메인 기준으로 등록되어 있는지 확인.
- **왜**: 미등록 상태로 prod 도메인에서 카카오 로그인 누르면 즉시 `redirect_uri_mismatch` 로 죽음. 발표 당일 도메인을 처음 트는 일이 없어야 함.
- **언제**: prod 도메인이 확정되고 Vercel 배포 시작되는 시점. 그 전엔 진행해도 의미 없음.
- **체크리스트**:
  1. Vercel 에 prod 도메인 연결됨
  2. 카카오 디벨로퍼스 → 카카오 로그인 → Redirect URI 에 `https://<prod-도메인>/auth/callback` 추가
  3. Supabase Auth → URL Configuration → Site URL 을 prod 도메인으로 설정
  4. Supabase Auth → URL Configuration → Redirect URLs 에 prod 도메인 추가
  5. dev 환경(`http://localhost:3001/auth/callback`) 도 함께 유지(삭제 X)
  6. 카카오 dev 앱 / prod 앱을 분리 운영하기로 했다면 prod 앱 키를 Vercel Production env 에 등록
