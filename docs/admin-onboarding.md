# 청풍 관리자 온보딩 가이드

> 목적: 청풍 담당자가 강화유니버스 대시보드를 직접 운영할 수 있도록 최소 운영 절차를 정리한다.
> 범위: 관리자 화면 사용, 프로젝트/에피소드/가게 관리, 사장님 코드 발급, 검수와 신고 대응.

## 1. 관리자 역할

관리자는 다음 작업을 맡는다.

- 프로젝트와 에피소드 등록 및 수정
- 참여 가게 등록과 QR token 관리
- 사장님 코드 발급과 재발급
- 공개 카드 검수
- 신고 카드 확인과 가림 처리
- 발표 전 시드 데이터 확인

관리자는 일반적으로 Supabase DB 콘솔을 직접 다루지 않는다. DB 직접 수정은 개발자 또는 기술 담당자가 맡는다.

## 2. 접속 경로

| 목적 | 경로 |
| --- | --- |
| 관리자 로그인 | `/admin/login` |
| 운영 홈 | `/admin` |
| 프로젝트/에피소드 관리 | `/admin/projects` |
| 가게/QR/사장님 코드 관리 | `/admin/shops` |
| 공개 검수 | `/admin/review` |
| 신고 처리 | `/admin/reports` |

배포 도메인이 바뀌면 위 경로 앞에 새 도메인을 붙여 사용한다.

## 3. 첫 관리자 계정 만들기

첫 관리자 계정은 Supabase Auth에서 만든 뒤 `app_metadata.role = "admin"`을 부여해야 한다. 이 작업은 개발자 또는 기술 담당자가 수행한다.

```sql
update auth.users
   set raw_app_meta_data = raw_app_meta_data || '{"role":"admin"}'::jsonb
 where email = '<관리자 이메일>';
```

주의:

- 관리자 비밀번호는 문서나 메신저 공개 채널에 남기지 않는다.
- 퇴사, 담당 변경, 계정 분실 시 관리자 계정을 즉시 회수하거나 재발급한다.
- 관리자가 아닌 계정에는 `admin` role을 부여하지 않는다.

## 4. 관리자 로그인

1. `/admin/login`에 접속한다.
2. 관리자 이메일과 비밀번호를 입력한다.
3. 로그인 후 `/admin` 운영 홈으로 이동하는지 확인한다.
4. 접근이 막히면 계정 role이 `admin`인지 확인한다.

## 5. 프로젝트와 에피소드 관리

### 프로젝트 등록

1. `/admin/projects`로 이동한다.
2. 새 프로젝트 등록 버튼을 누른다.
3. 카테고리를 선택한다.
4. 프로젝트 이름, slug, 설명, 공개 여부를 입력한다.
5. 진척도 기준을 설정한다.
6. 저장 후 `/projects/[slug]` 공개 화면에서 깨지지 않는지 확인한다.

카테고리는 2026 기준 4종만 사용한다.

| slug | 표시 이름 |
| --- | --- |
| `active_life` | 액티브 라이프 |
| `local_culture` | 로컬 문화 공동 창작 |
| `network` | 글로벌 & 로컬 네트워크 |
| `tech` | 테크 & 솔루션 |

`commons`, `world`, `policy`는 운영 분류로 되살리지 않는다.

### 에피소드 등록

1. 프로젝트 상세 또는 에피소드 관리 영역에서 새 에피소드를 만든다.
2. 이름, 날짜, 상태를 입력한다.
3. 상태는 `planned`, `in_progress`, `completed` 중 하나를 사용한다.
4. 저장 후 `/projects/[slug]`의 타임라인에 표시되는지 확인한다.

## 6. 가게와 사장님 코드 관리

### 가게 등록

1. `/admin/shops`로 이동한다.
2. 새 가게 등록 버튼을 누른다.
3. 가게 이름, 주소, 설명, 공개 여부를 입력한다.
4. QR token이 생성되었는지 확인한다.
5. `/entry/[qr_token]`으로 접속해 카드 작성 화면이 열리는지 확인한다.

### 사장님 코드 발급

1. `/admin/shops`에서 해당 가게 상세로 이동한다.
2. 사장님 코드 발급 또는 재발급을 실행한다.
3. 화면에 표시된 코드는 사장님에게 별도 안전한 채널로 전달한다.
4. 문서, GitHub, 공개 메신저에 평문 코드를 남기지 않는다.

운영 원칙:

- 사장님 코드는 가게별로 관리한다.
- 분실 시 기존 코드를 알려주지 않고 새 코드로 재발급한다.
- 잘못된 코드 5회 입력 시 1시간 잠금이 걸릴 수 있다.

## 7. 공개 검수 운영

1. `/admin/review`로 이동한다.
2. 공개 요청 또는 공개 상태인 카드를 확인한다.
3. 부적절한 사진, 개인정보, 초상권 문제가 있는지 확인한다.
4. 문제가 있으면 공개 해제 또는 가림 처리한다.
5. 처리 후 `/feed`, `/shops/[id]`, `/impact`에서 노출 상태가 반영되는지 확인한다.

검수 기준:

- 얼굴이 뚜렷한 사진은 `face_consent` 흐름을 확인한다.
- 개인 연락처, 주소, 민감 정보가 포함된 메모는 공개하지 않는다.
- 경쟁, 비난, 광고성 내용은 공개하지 않는다.

## 8. 신고 대응

1. `/admin/reports`로 이동한다.
2. 신고된 카드를 확인한다.
3. 문제 없음이면 신고 해제한다.
4. 공개하면 안 되는 카드면 가림 처리한다.
5. 긴급 개인정보 이슈는 개발자에게 storage 사진 정리까지 요청한다.

권장 운영:

- 신고 큐는 행사 기간 중 매일 확인한다.
- 최종 발표 전에는 발표에 쓰는 공개 카드만 한 번 더 확인한다.
- 삭제 요청은 `removed_at` 처리 후 공개 영역에서 사라졌는지 확인한다.

## 9. 크루 코드 운영

크루는 공용 코드로 `/crew/login`에 접속한다. 코드는 환경변수 `CREW_ACCESS_CODE`로 관리된다.

운영 원칙:

- 크루 코드가 외부에 노출되면 새 값으로 바꾸고 재배포한다.
- 코드 변경은 개발자 또는 배포 권한자가 처리한다.
- 크루가 할 수 있는 작업과 관리자가 해야 하는 작업을 회의에서 나눈다.

## 10. LLM 편지 운영

사장님 편지 화면은 AI가 첫 문장 또는 짧은 도입만 제안한다. 최종 편지는 사장님이 직접 수정해서 저장해야 한다.

운영 원칙:

- AI 초안은 그대로 보내지 않는다.
- API key가 없거나 무료 tier 한도에 걸리면 수동 작성 플로우로 진행한다.
- LLM provider는 현재 Gemini 기본이며, 추후 Anthropic API key가 준비되면 교체 가능하다.

## 11. 정기 운영 체크리스트

### 행사 전

- [ ] 프로젝트와 에피소드가 최신 상태인지 확인
- [ ] 가게 QR token이 실제 QR과 맞는지 확인
- [ ] 사장님 코드가 담당자에게 전달되었는지 확인
- [ ] `/entry/[qr_token]`에서 카드 작성 화면이 열리는지 확인
- [ ] `/impact`, `/feed`, `/shops` 공개 화면에 부적절한 카드가 없는지 확인

### 행사 중

- [ ] 새 카드가 `/collection`에 정상 생성되는지 확인
- [ ] 공개 카드가 `/feed`와 `/impact`에 반영되는지 확인
- [ ] 신고 큐를 확인
- [ ] 사장님 편지가 정상 저장되는지 확인

### 행사 후

- [ ] 최종 발표에 쓸 대표 카드와 숫자 확인
- [ ] 비공개 또는 신고 카드가 발표 화면에 보이지 않는지 확인
- [ ] 운영 중 발견한 문제를 `docs/feedback-questions.md` 결정 로그에 정리

## 13. 문제가 생겼을 때

| 문제 | 먼저 확인할 것 | 다음 조치 |
| --- | --- | --- |
| 관리자 로그인이 안 됨 | 이메일/비밀번호, admin role | 개발자에게 계정 role 확인 요청 |
| 사장님 로그인이 안 됨 | 가게 주소, 코드, 잠금 여부 | `/admin/shops`에서 코드 재발급 |
| QR 진입이 안 됨 | QR token, 가게 공개 여부 | `/admin/shops`에서 token 확인 |
| 카드가 공개 화면에 안 보임 | `is_public`, 신고/삭제 상태 | `/admin/review`, `/admin/reports` 확인 |
| AI 초안이 안 만들어짐 | API key, 무료 tier 한도 | 수동 작성 후 env 상태 확인 |
| 신고 카드가 계속 보임 | 처리 후 공개 화면 반영 여부 | 가림 처리와 캐시 새로고침 확인 |
